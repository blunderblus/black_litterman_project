## 맥락(Context)

BL의 투자자 전망(View, $Q$)은 **뷰 레지스트리 블록스택**(현 news·pattern 2뷰)으로 구성된다 — news(Gemini 감성), pattern(XGBoost 성장/이탈). relationship(거래관계 강도)은 방향 뷰가 아니라 현재상태=이동성이라 $\Sigma$ 이동성 슬롯으로 예약한다(E3a, [BL §5.1](../03-bl-model-design.md) 참조). IsolationForest 이상도(anomaly)는 방향 뷰가 아니라 $\Omega$ 신뢰도 변조 요인으로 들어간다([BL §5.4](../03-bl-model-design.md), 게인 $\gamma_{\text{anom}}=2.0$). 즉 모델 출력이 곧 BL 입력이므로, 모델이 과대평가되면 BL 사후수익·최적가중이 통째로 왜곡된다. 따라서 모델 검증의 정직성이 시스템 전체 타당성의 전제다. 본 ADR이 다루는 **신뢰도 캘리브레이션**은 XGBoost confidence·Gemini 감성 신뢰도(`gemini_confidence`)·이상도(anomaly) 등 $\Omega$로 들어가는 신뢰도 신호 전부를 대상으로 한다.

과거 토이 프로젝트는 시계열·라벨 처리에서 다수의 누수(leakage)를 안고 있었다.

- **As-is 실제 결함 1 — in-sample 평가(AUC 0.99급 허수의 1차 원인)**: 학습데이터로 그대로 평가했다(train/test 분리·CV 자체가 없음). 그 결과 보고된 성능(예: AUC 0.99급)은 일반화 성능이 아니라 암기(memorization)에 가깝다. **이 0.99급 허수의 직접 원인은 "분리·CV 부재(in-sample)"이며**, 이 수치는 미검증이고 신뢰할 수 없다.
- **As-is 실제 결함 2 — 라벨 look-ahead**: `bal_future_3m`(미래 3개월 잔액)이 라벨이면서 동시에 피처로 들어갔다. 미래정보가 피처에 섞이면 모델이 정답을 컨닝하게 되어 성능이 비현실적으로 부풀려진다.
- **As-is 실제 결함 3 — 스케일링 누수**: 추론배치에서 그 시점 데이터의 min/max로 $[-1,1]$ 정규화를 수행했다. 학습 기준이 아니라 실행 시점 분포로 스케일하므로 (a) 미래/배치 정보가 새고 (b) 실행마다 결과가 달라져 비교 불가능했다.
- **As-is 실제 결함 4 — confidence 하드코딩**: $0.85/0.65$ 등 신뢰도를 임의 상수로 박았다. BL의 전망 불확실성 $\Omega$는 이 신뢰도와 데이터신뢰도(DRI)에서 유도되는데, 근거 없는 상수는 $\Omega$를 왜곡한다.

> 참고: "셔플 random split"(분리는 하되 시간순서를 섞음)은 과거가 실제로 했다고 확인된 결함이 아니다. 그러나 분리를 도입하더라도 셔플하면 결함 1과 **동질의 낙관 편향이 재발**하는 별개 위험이므로, 대안 검토(아래)에서 명시적으로 기각한다.

격상판은 시점기반 검증, 미래정보 차단, 재현 가능한 스케일링, 실측 기반 신뢰도를 요구한다.

## 결정(Decision)

**시점분리·누수금지 학습 표준**을 전 모델링 단계의 의무 규약으로 채택한다.

1. **시점 기반 분리(temporal split)**: 모든 데이터는 시점 컬럼(`as_of`)을 기준으로 train < valid < test로 시간순 분리한다. 랜덤 셔플 split을 금지한다(시계열 누수 방지).
2. **walk-forward 백테스트(기본 구성 명시)**: 단일 holdout이 아니라 시간을 굴리며 반복 학습·평가하여 일반화·안정성을 측정한다. in-sample 평가는 전면 금지한다. 기본 구성(가설, 데이터 가용량에 따라 §9 민감도에서 조정):
   - 윈도우 방식: **expanding window**(월데이터 기반 $T<N$ 상황에서 학습량을 최대 확보; rolling은 분포 변화가 큰 구간 점검용 옵션).
   - test 길이 3개월, 1개월 step 전진, valid는 test 직전 3개월.
   - 라벨 horizon $h=3\text{M}$(`bal_future_3m`과 정합).
   - 구현 러너·평가지표는 [BL §9.1 Walk-forward 백테스트](../03-bl-model-design.md)를 준수한다.
3. **피처/라벨 시점 엄격 분리 + embargo**: 라벨은 미래 구간($t{+}1..t{+}h$)에서, 피처는 기준시점 이전 정보에서만 생성한다. `bal_future_*`처럼 미래를 담은 변수의 **피처 사용을 차단**한다(라벨 전용). 모든 피처에 "기준시점 $t$에서 알 수 있었는가"(point-in-time) 검사를 적용한다. 라벨 horizon $h$ 때문에 **train 마지막 시점과 valid/test 시작 사이에 $h$개월 embargo(purge) 구간을 강제**한다(미적용 시 빌드 실패; 데이터계층 강제는 [데이터 파이프라인 §5.1·§5.4](../02-data-pipeline.md), 금지 피처 목록은 [§3.2.1 `train_set` 스키마](../02-data-pipeline.md)).
4. **고정 스케일러(fitted-on-train)**: **스케일링 방식 자체는 교체하지 않는다(현행 변환을 유지)**. 핵심은 **fit 범위를 train으로 고정**하는 것이다. 스케일러는 train 구간에서만 fit하고 그 파라미터(평균/표준편차/clip 범위 또는 min/max — 변환에 대응)를 저장(ADR-0002에 따라 pickle 금지, JSON/Parquet 파라미터로 저장)한 뒤 valid/test/추론에 동일 적용한다. 추론 시점 min/max 재계산을 금지한다(재현성·비교가능성 확보). 저장 파라미터 표기는 [데이터 파이프라인 §5.3](../02-data-pipeline.md)과 일치시킨다.
5. **신뢰도 캘리브레이션(calibration)**: confidence 하드코딩을 금지하고, 검증셋 기반 실측 캘리브레이션(reliability/Platt·isotonic 등)으로 각 모델 확률 신뢰도(`confidence_*`, `gemini_confidence`)를 보정한다. 보정된 신뢰도 $\text{conf}$는 [BL §5.4](../03-bl-model-design.md)의 $\Omega$ 결합식
   $$\Omega_{kk} = (P\,\tau\Sigma\,P^{\top})_{kk}\times\frac{1}{\text{DRI}_i^2}\times\frac{1-\text{conf}_i}{c_{\text{cal}}}\times(1+\gamma_{\text{anom}}a_i)$$
   의 $\frac{1-\text{conf}_i}{c_{\text{cal}}}$ 항으로 대입한다($\Omega \propto 1/\text{DRI}^2$ 의미론 보존). 캘리브레이션 상수 $c_{\text{cal}}$과 $\Omega$ 하한 $\Omega_{\text{floor}}$는 검증셋에서 산정한다.
6. **누수 감사(leakage audit)**: 학습 파이프라인에 자동 점검을 둔다 — train/valid/test 시점 겹침 검사(+embargo $h$ 격리 확인), 금지 피처(미래변수) 사용 차단, 스케일러 fit 범위 검증. 추가로:
   - **캘리브레이션 누수 차단**: 확률 캘리브레이션은 train 내부 분할(또는 nested CV)에서만 fit하고, test/추론 구간 데이터로 보정하지 않는다(검증셋이 walk-forward test 구간과 겹치면 미래정보 누수).
   - **집계 피처 누수 차단**: 타깃 인코딩·그룹/시계열 집계 통계 피처는 fit 범위를 train으로 한정한다(fold/시점 누수 방지).
   위반 시 빌드 실패. 데이터계층 게이트는 [데이터 파이프라인 §5.4 누수 감사](../02-data-pipeline.md)와 정합시킨다.

## 근거(Rationale)

모델 출력이 BL 입력이라는 본 시스템 구조상, 부풀려진 성능은 단순 보고 오류가 아니라 자원배분 의사결정을 직접 오도한다. 시점분리·walk-forward는 시계열 예측에서 일반화 성능을 정직하게 추정하는 표준이며, 고정 스케일러·미래변수 차단·캘리브레이션은 각각 과거의 구체적 결함(스케일 누수, look-ahead, 신뢰도 하드코딩)을 1:1로 해소한다. **과거 AUC 0.99급 허수의 1차 원인은 in-sample 평가(분리·CV 부재, 결함 1)이며**, 셔플 random split은 "설령 분리를 도입하더라도 시계열을 섞으면 동일 낙관 편향이 재발하는" 또 다른 누수 경로로서 함께 차단한다.

대안 비교(채택 표준 vs 검토 후 기각 후보):

| 기준 | 시점분리·누수금지 표준 (채택) | 단순 랜덤 split (기각 후보) |
|---|---|---|
| 시계열 누수 방지 | 높음(시간순 분리+embargo) | 낮음(미래→과거 누수) |
| 일반화 성능 추정 정직성 | 높음(walk-forward) | 낮음(낙관 편향) |
| 실운영 분포 변화 대응 평가 | 가능(expanding/rolling window) | 불가 |
| look-ahead(bal_future) 차단 | 명시적 차단 | 미보장 |
| 스케일링 재현성 | 높음(고정 스케일러) | 낮음(배치 의존 가능) |
| 신뢰도 정합($\Omega$ 연결) | 실측 캘리브레이션 | 미해결 |
| 구현 복잡도 | 중간(분할·감사 필요) | 낮음 |
| 결과 신뢰도 | 높음 | 낙관편향으로 인한 허수 위험(과거 in-sample 0.99와 동질) |

단순 랜덤 split은 구현이 쉽지만 시계열에서 미래 표본이 학습에 섞여 낙관 편향을 만든다 — 이는 과거 in-sample 0.99 허수와 **동질의 위험**(분리는 했으나 셔플 시 재발)이다. 본 프로젝트는 "성능 수치를 단정하지 말 것"을 규약으로 두고 있으며, 그 정직성을 보장하는 방법이 시점분리·누수금지 표준이다.

## 결과(Consequences)

긍정적:
- 보고 성능이 일반화 성능에 수렴하여 BL 입력($Q$, $\Omega$)의 신뢰성이 올라가고, 자원배분 의사결정의 타당성이 확보된다.
- 고정 스케일러로 실행 간 결과가 재현·비교 가능해진다(과거 "실행마다 다른 정규화" 해소).
- 캘리브레이션된 신뢰도가 [BL §5.4](../03-bl-model-design.md)의 결합식을 통해 DRI 기반 $\Omega$와 정합되어 BL Ω 의미론($\Omega \propto 1/\text{DRI}^2$, 데이터 빈약 고객 뷰 불신)이 일관되게 구현된다.
- 누수 감사가 회귀 가드로 작동해 동일 결함 재발을 차단([데이터 파이프라인 §5.4](../02-data-pipeline.md)).

부정적:
- **연산 증가**(반복 학습): GPU 가속([ADR-0001](./ADR-0001-compute-backend.md))으로 완화된다(격상은 속도만 차이, 동일 수치).
- **가용 학습량 감소**(특히 월데이터 기반 $T<N$ 및 embargo로 인한 추가 손실): 이는 GPU로 완화되지 않는 별도 문제이며, expanding window·수축공분산([BL §3.2](../03-bl-model-design.md))으로 대응한다.
- 미래변수 차단으로 일부 강력해 보였던 피처를 버려야 해, 표면상 성능이 과거 대비 "낮아" 보일 수 있다(실은 정직해진 것).
- 캘리브레이션·감사 파이프라인 구축에 초기 비용이 든다.

후속작업:
- [ ] `as_of` 기반 train/valid/test 시점분리 유틸리티와 walk-forward(expanding 기본/rolling 옵션) 백테스트 러너 구현(embargo $h$ 격리 포함). 데이터계층 강제는 [데이터 파이프라인 §5 시점정합](../02-data-pipeline.md).
- [ ] point-in-time 피처 생성 규약 + 금지 피처(`bal_future_*` 등) 차단 목록·검사 구현([데이터 파이프라인 §3.2.1 `train_set` 스키마·§5.4](../02-data-pipeline.md)).
- [ ] train-fit 스케일러의 파라미터 저장/로드(ADR-0002 준수) 및 추론 재계산 금지 가드([데이터 파이프라인 §5.3](../02-data-pipeline.md)).
- [ ] 검증셋 기반 확률 캘리브레이션(Platt/isotonic 등)과 신뢰도→$\Omega$ 연결 로직 구현([BL §5.4](../03-bl-model-design.md)의 $\frac{1-\text{conf}}{c_{\text{cal}}}$ 항, $c_{\text{cal}}$·$\Omega_{\text{floor}}$ 검증셋 산정).
- [ ] 누수 감사(시점 겹침·embargo·금지피처·스케일러 fit 범위·캘리브레이션/집계 누수) 자동 점검을 빌드 게이트로 추가하고 [BL §9](../03-bl-model-design.md)·[데이터 파이프라인 §5.4](../02-data-pipeline.md)에 절차 명시.

## 대안(Considered Alternatives)

- **단순 랜덤 split(셔플 후 train/test)**: 정량 비교·기각 사유는 위 근거 표 및 본문 참조. 요지는 시계열 셔플이 과거 in-sample 0.99와 동질의 낙관 편향을 재발시키며 정직성 규약과 충돌한다는 것이다.
- **단일 holdout(시간순 1회 분할)만 사용**: 랜덤보다 낫고 누수도 줄지만 분포 변화에 대한 안정성을 측정하지 못한다. 따라서 최종 검증 기준으로는 기각하되, **walk-forward의 부분집합(빠른 점검·CI 스모크용)으로만 인정**한다(이 점이 본 대안의 고유 가치).
- **confidence 하드코딩 유지(상수 신뢰도)**: 매력 요인은 단순성(추가 검증·보정 파이프라인 불필요)뿐이다. 그러나 근거 없는 상수가 $\Omega$ 결합식의 $1-\text{conf}$ 항을 왜곡해 BL 결과를 정량적으로 오도하므로(과신 시 $\Omega\to$ 과소 → 사후 폭주), 실측 캘리브레이션으로 대체하여 기각.
